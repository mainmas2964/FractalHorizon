Введение

Идея

Мир делится на чанки (например, 16×16×256). Строим статический BVH по AABB чанков. Для трассировки луча сначала фильтруем чанки через BVH, затем внутри каждого попавшего чанка выполняем 3D‑DDA (voxel traversal). Динамические объекты держим в отдельном TLAS и обрабатываем отдельно.

Преимущества
  • Минимальные глобальные проверки.
  • Простая и быстрая локальная проверка внутри чанка.
  • Масштабируемость на миллионы блоков.

Алгоритм

Шаг 1 — Поиск кандидатов в BVH по чанкам

Обходить BVH итеративно, проверяя пересечение луча с AABB узлов. Для каждого листа (чанка) получить интервал параметров луча внутри AABB и добавить чанку в список кандидатов. Обрабатывать кандидатов в порядке возрастания t_enter.

Шаг 2 — Инициализация 3D‑DDA внутри чанка

Перевести точку входа луча в локальные координаты чанка. Определить начальный воксель i. Вычислить направление шага по каждой оси step_k. Рассчитать tDelta_k и tMax_k. Обработать случаи нулевой компоненты направления, задав бесконечные шаги по соответствующей оси.

Шаг 3 — Цикл DDA по вокселям

Пока текущий воксель внутри границ чанка и текущее t не превысило t_exit:

  • Если воксель твёрдый — вернуть попадание и остановиться.
  • Иначе выбрать ось с минимальным tMax, шагнуть по ней, обновить t и tMax.
Шаг 4 — Переходы между чанками

При выходе из AABB чанка продолжить с следующего чанка по отсортированному списку кандидатов. Пропускать полностью пустые чанки с помощью флага empty без запуска DDA.

Ключевые формулы

Пересечение луча с AABB
\[ \mathbf{r}(t)=\mathbf{o}+t\mathbf{d} \]
\[ t_{k}^{\min} = \dfrac{b_{\min,k} - o_k}{d_k},\quad t_{k}^{\max} = \dfrac{b_{\max,k} - o_k}{d_k} \]
\[ t_{\text{enter}} = \max_k \min(t_k^{\min},t_k^{\max}),\quad t_{\text{exit}} = \min_k \max(t_k^{\min},t_k^{\max}) \]

Если \(t_{\text{enter}} \le t_{\text{exit}}\), луч пересекает AABB в интервале \([t_{\text{enter}}, t_{\text{exit}}]\).

Параметры 3D‑DDA

Определения
\[ \text{step}_k = \operatorname{sign}(d_k) \]
\[ tDelta_k = \dfrac{1}{|d_k|},\quad tMax_k = \dfrac{(i_k + \mathbb{1}_{d_k>0}) - p_k}{d_k} \]

В цикле выбираем ось с минимальным \(tMax_k\) и выполняем шаг по ней, обновляя \(i_k\), \(t\) и \(tMax_k\). При \(d_k = 0\) задавайте \(tDelta_k = +\infty\) и \(tMax_k = +\infty\).

Практические рекомендации и оптимизации

  • Чанки как листья BVH — хранить AABB на уровне чанков.
  • Empty mask и уровни пустоты — бит‑маски для быстрого пропуска пустых областей.
  • Fat AABB — небольшой запас для устойчивости к численным ошибкам.
  • Пакетная трассировка лучей — группировка лучей по направлению/области.

Минимальные фрагменты кода

// p = o + d * t_enter
p = o + d * t_enter;
i = floor(p); // начальный воксель (в локальных координатах чанка)
step = sign(d);
tDelta = { invAbs(d.x), invAbs(d.y), invAbs(d.z) };

// tMax: расстояние по параметру t до первой границы вокселя по каждой оси
tMax.x = ( (i.x + (d.x > 0 ? 1 : 0)) - p.x ) / d.x; // handle d.x == 0 -> +inf
tMax.y = ( (i.y + (d.y > 0 ? 1 : 0)) - p.y ) / d.y;
tMax.z = ( (i.z + (d.z > 0 ? 1 : 0)) - p.z ) / d.z;

// если d.k == 0: tDelta.k = +inf, tMax.k = +inf

t = t_enter;
while (insideChunk(i) && t <= t_exit) {
    if (isSolid(i)) {
        return hit;
    }
    if (tMax.x < tMax.y && tMax.x < tMax.z) {
        i.x += step.x; t = tMax.x; tMax.x += tDelta.x;
    } else if (tMax.y < tMax.z) {
        i.y += step.y; t = tMax.y; tMax.y += tDelta.y;
    } else {
        i.z += step.z; t = tMax.z; tMax.z += tDelta.z;
    }
}

Заключение

Комбинация статического BVH по чанкам и 3D‑DDA внутри чанка даёт оптимальный баланс для воксельных миров: глобальная фильтрация остаётся дешёвой, а локальная проверка — максимально простой и быстрой.
